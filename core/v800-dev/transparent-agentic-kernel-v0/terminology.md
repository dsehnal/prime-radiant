# Terminology Mapping

## Core primitives and actions

| Membrane-era term | Current term    | Notes                                                                                                                                                         |
| ----------------- | --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Record (≀)**    | **Context**     | “Record” stressed written retention; “Context” is softer and more transparent, but still means accumulated retained structure.                                |
| **Membrane (⊙)**  | **Perspective** | “Membrane” stressed active boundary/interface; “Perspective” is more readable, but slightly less vivid.                                                       |
| **Unfold**        | **Interpret**   | “Unfold” carried symmetry with Fold; “Interpret” makes the act more explicit and less mystical.                                                               |
| **Fold**          | **Integrate**   | “Fold” was elegant and compressed; “Integrate” is clearer in plain language. Important correction retained: `Integrate :: Context -> Perspective -> Context`. |

# Clean one-line correspondence

If you want the shortest mapping:

**Record / Membrane / Unfold / Fold / Gauge / Shield**
became
**Context / Perspective / Interpret / Integrate / Lens / structured remainder**,
with the major new clarification that the cycle yields both **resolution** and **remainder**.

# Deep structural changes, not just renamings

Three changes were more than vocabulary:

1. **Fold was corrected explicitly**

   * from an intuitively unary description
   * to the true two-argument structure:
     `Integrate :: Context -> Perspective -> Context`

2. **Remainder was split conceptually**

   * into **resolution** and **remainder**
   * which made “residue” unnecessary

3. **Gauge language was simplified**

   * from shadows/gauges/instrument
   * to process/lenses/shared context

# Best compact glossary

If you want a minimal glossary block for the repo:

* **Context** = retained structure
* **Perspective** = current operative orientation
* **Interpret** = expose latent structure in context
* **Integrate** = rewrite context through perspective
* **Resolution** = what now integrates cleanly
* **Remainder** = what cannot yet be honestly integrated
* **Lens** = a way of investigating the process
* **Shared context** = writable context across multiple perspectives
