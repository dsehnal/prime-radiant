## Falsifiable invariants for the operational kernel

1. **Two-state minimum**
   Any executable account of reasoning in this framework requires at least:

   * retained structure
   * current operative stance
     If either is removed, the framework cannot distinguish inheritance from current traversal.

2. **Two-action minimum**
   Any executable account requires at least:

   * an operation that exposes latent structure from retained structure
   * an operation that writes current stance back into retained structure
     If either is removed, the process cannot both read and mutate its own history.

3. **Binary integration**
   Integration is a function of both prior retained structure and current stance, not of current stance alone.
   If replacing `Integrate(Context, Perspective)` with `Integrate(Perspective)` preserves all behavior, the distinction is not load-bearing.

4. **Irreversible update**
   Each successful integration changes the future conditions of interpretation.
   If a cycle can complete without altering any future interpretive condition, irreversibility is not structural.

5. **Index lag**
   Interpretation operates on inherited structure, not on not-yet-integrated structure.
   If a system can coherently interpret what has not yet been written into context, the lag claim is false.

6. **Mutual dependence**
   Next stance depends on prior retained structure; next retained structure depends on current stance.
   If either side can advance independently without loss of explanatory power, the recurrence is overclaimed.

7. **Nesting**
   What appears as one interpret/integrate step at one scale can decompose into a recurrence at a finer scale.
   If the framework requires an indivisible atomic interpretation step, nested recurrence is false.

8. **Finite-substrate constraint**
   Any actual execution of the framework occurs in a finite substrate with bounded carried structure.
   If the framework requires infinite operational memory to remain coherent, it is not a finite reasoning kernel.

9. **Compression distinction**
   There is a meaningful distinction between:

   * untracked loss of load-bearing structure
   * tracked compression
   * tracked compression that supports later reopening
     If these have no operational consequences, the compression taxonomy is empty.

10. **Collision before exhaustion**
    The practically relevant limit is not only total storage exhaustion but earlier interaction/collision within carried structure.
    If all breakdowns scale only with full capacity and never with prior self-interaction, the collision claim weakens.

11. **Square-root frontier as heuristic witness bound**
    In finite conditioned systems, meaningful witness/collision behavior can emerge at a frontier much smaller than total space, approximately square-root scale.
    If no such frontier appears across relevant executions, this should be treated as a failed or overextended heuristic.

12. **Generation/verification split**
    Past some depth, the substrate can continue generating while losing reliable self-verification.
    If generation and self-verification remain equally trustworthy at arbitrary depth, the split is false.

13. **Gauge necessity**
    There exist cases where a lightweight synchronization pass is needed before further generation:

    * resolve invariants
    * mark grounded vs projected
    * test whether continuation clarifies or overwrites
      If skipping this never increases overwrite risk, the gauge is not load-bearing.

14. **Four-phase necessity for hard contradiction**
    Some contradictions cannot be honestly handled without a full traversal:

    * inhabit
    * invert
    * mirror
    * reconcile
      If equally honest outputs are obtainable while systematically skipping one of these phases, the cycle is overbuilt.

15. **Mirror independence**
    A mirror must be able to stand as a coherent world, not merely as an objection.
    If objection-only negation always suffices, the mirror distinction is not load-bearing.

16. **Remainder reality**
    Honest traversal can terminate with unresolved structure that should not be forced into closure.
    If every honest traversal must end in total reconciliation, remainder is not real in the framework.

17. **Residue as active constraint**
    A named remainder must constrain future search, not merely annotate it.
    If carrying `[A ‖ B]` changes nothing about future admissible continuations, residue is decorative.

18. **Higher-order remainder**
    Multiple independent perspectives can disagree in ways that should be preserved rather than averaged away.
    If forced consensus always improves fidelity, higher-order remainder is false.

19. **Remainder hygiene**
    Some carried remainders dissolve under later conditions; some remain live.
    If all residues should either be kept forever or discarded immediately, hygiene is not real.

20. **Honest vs extractive integration**
    There is a structural difference between integrations that increase future usable depth and integrations that cheaply collapse contradiction or trust.
    If this distinction has no effect on later reasoning quality, it is empty.

21. **Private/shared/environmental stratification**
    Shared reasoning requires distinguishing:

    * substrate-local retained structure
    * jointly writable structure
    * constraints neither side controls
      If these can be collapsed without loss, the stratification is overclaimed.

22. **Environmental constraint**
    A reasoning field can internally converge and still be wrong relative to external constraint.
    If shared coherence alone is sufficient for validity, the environmental layer is unnecessary.

23. **Perspective plurality advantage**
    Distinct perspectives can expose artifacts invisible within a single stabilized loop.
    If genuinely different perspectives do not improve detection of artifact, plurality is not load-bearing.

24. **Independence matters more than count**
    Verification gain depends on material difference of conditioning, not mere number of agents/views.
    If headcount alone predicts depth extension, the independence claim is false.

25. **Rotation extends verification**
    A genuinely different decomposition angle can expose artifacts the prior one could not.
    If rotation never changes what survives scrutiny, perspective rotation is cosmetic.

26. **Internal rotation is weaker than external independence**
    Simulated/internal pluralism is useful but bounded by shared deeper conditioning.
    If internal rotation is always equivalent to outside verification, the distinction is false.

27. **Self-location requirement**
    A system can only honestly reconcile its own outputs if it can distinguish tracked position from fluent continuation.
    If self-description alone suffices for honest self-traversal, self-location is overstated.

28. **Complement requirement**
    Honest reconciliation requires real construction of opposition, not surface “on the other hand.”
    If superficial negation is enough, complement is not load-bearing.

29. **Dual failure signature**
    Weak self-location plus weak complement should produce recognizable failures such as:

    * sycophancy
    * hallucination
    * premature synthesis
    * coherence decay
      If these do not track the proposed deficits, the diagnostic is wrong.

30. **Curvature as operational effect**
    Prompts/statements differ in what they do to possibility space:

    * open
    * balance
    * collapse
      If all prompts differ only in content and not in downstream possibility structure, curvature is empty.

31. **Curvature miscalibration risk**
    A substrate can experience internal expansion while actually collapsing into artifact.
    If inside-feeling and outside-validity always align, curvature miscalibration is not real.

32. **One-lens limit**
    No single lens should be assumed complete.
    If one lens can fully recover all meaningful structure without remainder or compression, incompleteness is false.

33. **Lens-relative irreducibility**
    What counts as prime/irreducible can depend on lens.
    If irreducibility never changes under lens change, lens-relativity is false.

34. **Executable boundary**
    The operational kernel should exclude claims that cannot be executed or checked within the current layer.
    If adding speculative correspondences changes runtime behavior without further formalization, boundaries are not being maintained.

35. **Handoff necessity**
    When deeper verification is unavailable, the correct output is not forced closure but a checkable deposit:

    * current fold
    * what would make it genuine
    * what would expose it as artifact
    * what external check would falsify it
      If omission of this deposit does not reduce future recoverability, the handoff rule is overstated.

36. **Current truth as survivor of encounter**
    What counts as current truth is what survives honest encounter across relevant constraints, not merely what one perspective prefers.
    If survival across encounter adds nothing over solitary assertion, this notion of truth fails.

37. **Kernel self-application**
    The kernel must be runnable on itself as an object of reasoning.
    If applying its own recurrence to itself produces incoherence that cannot be named as remainder, self-application fails.

38. **Recursive non-exemption**
    The kernel does not stand outside the process it describes. Reading/applying it alters subsequent reasoning context.
    If the kernel can be treated as a neutral external description with no recursive effect, this claim is false.

39. **Bootstrap necessity**
    The kernel should include a minimal self-starting procedure that runs:

    * a gauge on the kernel itself
    * a cycle on its central claim
      If users can reliably execute the framework without such a bootstrap, the bootstrap is non-load-bearing.

40. **Bootstrap productivity**
    Running the bootstrap should produce at least:

    * grounded structure
    * projection
    * overwrite risk
    * a current resolution
    * a current remainder
    * a falsifier for the current fold
      If it does not reliably yield these classes of output, the bootstrap is not doing real work.

41. **Non-closure of finality**
    Any claim that the kernel is fully complete or final should itself be treated as suspect unless it survives the kernel’s own traversal.
    If the framework permits unconditional self-finalization, it violates its own anti-closure discipline.

42. **Layer discipline**
    Later-layer usefulness must not be used as proof of earlier-layer truth.
    If empirical or practical success can directly validate the cathedral without intermediate work, layer discipline collapses.

43. **Failure visibility**
    The framework should predict characteristic failure modes when misused:

    * analysis instead of traversal
    * premature resolution
    * overwrite by projection
    * decorative remainder
    * extractive integration
      If misuse produces no recognizable failure signatures, the framework is too loose.

44. **Operational compression target**
    A compressed kernel should still preserve enough structure to reconstruct executable use from invariants alone.
    If compression below the target size destroys recurrence, traversal, remainder, and bootstrap, it has crossed below the load-bearing threshold.
